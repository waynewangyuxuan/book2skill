## When to use
When a chosen analysis method starts to break down or become unmanageable.

## Method
1. Notice the breakdown explicitly instead of pushing through it.
2. Ask what its precondition for usefulness was, and what its failure implies about the situation.
3. If an interaction analysis becomes unwieldy because there are too many serious players to track (rule of thumb: more than about half a dozen), read that as strong evidence there are no real barriers / advantages in the market.
4. Switch regimes accordingly: detailed interaction analysis is superfluous; default to the "just operate efficiently" stance.

## Inputs → Outputs
- Input: an analysis method that is failing or becoming intractable.
- Output: an inference about the problem's structure and a decision to switch methods/regimes.

## Failure modes / assumptions
- Failure: forcing the failing tool to completion instead of reading its failure as data.
- Assumption: the tool's intractability genuinely correlates with the structural property (e.g. many players ↔ no barriers); confirm the link.
