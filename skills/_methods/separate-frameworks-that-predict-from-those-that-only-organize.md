## When to use
When an analytic apparatus (matrices, trees, scoring rubrics, models) is impressive-looking but, on inspection, the authors concede it "cannot be solved" or that the discipline is "imprecise."

## Method
1. Ask the apparatus to make a *falsifiable* prediction about a specific case. If it can't — if it only frames inputs and hands the actual judgment back to the user — it is an organizing tool, not a predictive one.
2. Recognize that an organize-only tool can be neither right nor wrong about outcomes; all the real prediction lives in the user's judgment that the tool merely arranges.
3. This isn't a reason to discard it — organizing is valuable — but it *is* a reason not to let the formalism's polish stand in for evidence or accuracy.

## Inputs → Outputs
- In: an analytic framework and its claims.
- Out: a classification (predicts vs organizes) and an explicit note of where the load-bearing judgment actually sits.

## Failure modes / assumptions
- Some frameworks predict in narrow regimes and organize elsewhere; classify per use, not globally.
- Don't use "it only organizes" as a slur — organizing complexity is often the most useful thing available. The error is mistaking it for prediction.
