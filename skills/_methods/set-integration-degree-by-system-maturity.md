## When to use
Deciding (or evaluating) how much of a value chain an actor should own/build vs buy, especially as its industry matures.

## Method
1. Assess the maturity of the supplier ecosystem for each function.
2. Model the optimal integration degree as a function of that maturity: integrate when external suppliers are unreliable or subscale; de-integrate as they mature and specialists achieve scale economies the integrated firm cannot match in any single component.
3. Treat "still doing it in-house" in a mature industry as a recurring cost-disadvantage signature, and expect the same maturation arc to recur across otherwise-unrelated industries.

## Inputs → Outputs
- In: a value chain + the maturity/scale of external suppliers per function.
- Out: a maturity-conditioned make-vs-buy recommendation and a flag on stale in-house functions.

## Failure modes / assumptions
- If a function is itself a genuine source of advantage/barrier, keeping it in-house can remain correct even in a mature industry.
- Switching costs, secrecy, or quality control can justify integration beyond the pure cost calculus.

## Worked example (illustrative, single)
A hardware maker that designed its own commodity components lost its edge once specialist component makers matured and reached comparable scale — the same arc that earlier turned a fully vertically-integrated manufacturing plant from an asset into a liability. Recognizing the maturation pattern flags the in-house functions to divest.
