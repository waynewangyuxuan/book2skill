# Recognize relocation is not resolution under conservation

## When to use
When a proposed fix works by transferring a conserved burden somewhere else and is being credited as a solution.

## Method
1. Confirm the burden is conserved — it must land somewhere.
2. Trace where the fix sends it.
3. Recognize that relocation reproduces the same problem at the new holder; it is not resolution.
4. Name the next holder and ask why they would accept it — this usually exposes infeasibility or a just-deferred crisis.
5. Track the aggregate, not the local relief.

## Inputs -> Outputs
Inputs: a relocation-style fix; the conserved burden. Outputs: the identified next holder; relocation-vs-resolution verdict.

## Failure modes / assumptions
- Relocation can buy legitimate time — distinguish "no resolution" from "useful delay."
- Requires the burden to be genuinely conserved (not dissipable).

## Worked example
A plan to fix one country's imbalance by shifting it onto a partner is shown to merely move the buildup, recreating the same problem next door.
