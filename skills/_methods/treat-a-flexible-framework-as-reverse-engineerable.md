## When to use
When a flexible, many-knobbed method is being used to justify a conclusion you (or a presenter) may already favor.

## Method
1. Count the subjective levers the method exposes; the more there are, the more it can be tuned to a foregone answer.
2. Recognize that such a method's output confirms a prior rather than testing it.
3. Pre-commit the key inputs before running it, so the result is not reverse-engineered.
4. Show a range across reasonable input settings rather than one tuned point.
5. Cross-check the conclusion externally, with a method that does not share its flexibility.

## Inputs to Outputs
- Inputs: a flexible method; its subjective levers; pre-committed inputs.
- Outputs: a range-based, externally-checked result instead of a tuned point.

## Failure modes / assumptions
- Treating a tuned output as independent evidence.
- Choosing inputs after seeing which give the desired answer.

## Worked example (illustrative, single)
A valuation that can produce any number from plausible inputs is shown as a wide range and cross-checked, rather than quoting the single flattering figure.
