# Reject an explanation that fits an outcome and its opposite

## When to use
When a fixed trait or standing condition is used to explain a result, and you suspect the same trait would have been used to explain the reverse.

## Method
1. State the explanation and the outcome it is currently credited with.
2. Imagine the opposite outcome; ask whether the same explanation would have been offered for it.
3. If yes, the explanation is unfalsifiable and carries zero discriminating content — demote it to narrative.
4. Replace it with a candidate that predicts one outcome and not the other.

## Inputs -> Outputs
Inputs: a trait-based explanation; the outcome. Outputs: a keep/demote verdict plus a discriminating replacement candidate.

## Failure modes / assumptions
- A trait can be a real cause and still get over-applied; the test demotes the lazy version, not the trait per se.

## Worked example
A national-character trait credited for success is shown to have been blamed for failure when the cycle turned, so it is discarded as a label.
