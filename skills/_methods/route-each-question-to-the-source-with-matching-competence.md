## When to use
Deciding whom to ask for a given piece of information across a built network.

## Method
1. Map your network by what each source can actually answer, not by headcount.
2. Distinguish sources good at explaining the past from those good at forecasting the future and route by question direction.
3. Prefer independent outside sources over constrained insiders, who are both restricted from disclosing and biased when they do.
4. Send each question to the source whose competence matches it; the value is routing precision.

## Inputs → Outputs
- Inputs: a source roster annotated by domain and direction.
- Outputs: a question→source routing map.

## Failure modes / assumptions
- Asking a generalist a specialist question and getting noise.
- Treating a large contact list as edge when retrieval precision is the real asset.

## Worked example (illustrative, single)
A backward-looking question goes to a former operator; a forward-looking one goes to an independent industry specialist, each matched to competence.
