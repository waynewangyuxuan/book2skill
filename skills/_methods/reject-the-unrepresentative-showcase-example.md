# Reject the unrepresentative showcase example

## When to use
When an argument leans on a vivid, frequently-cited example presented as typical.

## Method
1. Identify the relevant dimension on which the example is supposed to be representative.
2. Locate the example on that dimension's distribution.
3. If it sits in the tail, flag it as an outlier that cannot bear a general claim.
4. Demand a median-case example or a full distribution instead.

## Inputs -> Outputs
Inputs: the cited example; the relevant distribution. Outputs: representative vs outlier verdict.

## Failure modes / assumptions
- An outlier can still be a valid existence proof — distinguish "exists" from "typical."
- The relevant dimension must be the one the argument actually needs.

## Worked example
A famous "counterexample" country is shown to be a tail case on the relevant variable, so it cannot refute the general pattern.
