## When to use
When an analysis simplifies by stipulating that agents will "naturally do the right thing" / execute the jointly-optimal plan once it's identified — especially if an earlier part of the same framework established that defection is the default.

## Method
1. Locate the simplifying assumption that makes the analysis tractable.
2. Ask whether that assumption *is* the hard problem in disguise. Assuming agents won't defect, in order to analyze how cooperation works, is circular.
3. Cross-check against the framework's own prior claims: does the cooperative section quietly contradict the section that proved defection is the equilibrium?
4. Demand the missing mechanism: *why* won't agents defect? Enforcement, repetition, side-payments? If the framework only stipulates compliance rather than explaining it, its cooperative conclusions are unsupported.

## Inputs → Outputs
- In: an analysis + its key simplifying assumption.
- Out: whether the assumption smuggles away the central difficulty, any internal contradiction, and the enforcement mechanism that's missing.

## Failure modes / assumptions
- Idealized assumptions are legitimate for *bounding* an outcome ("here's the best case"); the error is presenting the idealized case as achievable without addressing incentives.
