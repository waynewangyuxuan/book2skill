# Falsify a cause by counter-case and cross-regime invariance

## When to use
When a popular explanation is offered for an outcome and you need to test whether it is load-bearing or merely a coincident feature.

## Method
Apply whichever disconfirmation probe the evidence allows:
1. **Absent-cause.** Find instances of the same outcome where the alleged cause did not exist; a real cause is present in all instances of the class.
2. **Sign-flip.** Find a period when the same fixed trait was invoked to explain the opposite outcome; a cause that "explains" both X and not-X explains neither.
3. **Cross-section.** Hold the outcome fixed and scan peers; if peers show the outcome while varying freely on the alleged cause, the cause is dispensable.
4. **Natural experiment.** Find when the lever was actually pulled; if the outcome moved the wrong way, the story is falsified.
5. **Hard case.** Pick the member that should most disconfirm; if even it shows the outcome, a deeper common driver is implicated.
6. **Channel-disable.** Build the extreme case where the cited channel cannot operate; if the outcome still holds, that channel is not the driver.

## Inputs -> Outputs
Inputs: an outcome + a proposed cause. Outputs: a verdict (load-bearing / dispensable) plus the disconfirming case(s).

## Failure modes / assumptions
- Prefer convergence of multiple probes over a single anecdote.
- Distinguish a confounded null (the lever was offset by a counter-move) from a genuine wrong-sign result.

## Worked example
A trait-based explanation is discredited by showing the same trait was blamed for the opposite outcome a generation earlier and that peers vary freely on it.
