# Separate the inevitable endpoint from the choosable path

## When to use
When an outcome is forced by an identity (a quantity must revert) but the debate is stuck on whether it will happen.

## Method
1. Establish, from the identity, that the endpoint is certain.
2. Separate the certainty of the endpoint from the choice of route.
3. Enumerate the arithmetically-valid paths to the same endpoint.
4. Rank them by welfare/cost.
5. Frame the decision as choosing the least-cost path to a forced destination — and, when trading, separate the priced-in certain destination from the uncertain path and bet on which channel the configuration selects.

## Inputs -> Outputs
Inputs: the forced endpoint; the set of valid paths; a cost ranking. Outputs: the least-cost path (policy) or the path bet (trade).

## Failure modes / assumptions
- Misjudging the endpoint as forced when it is only likely.
- Paths may interact; ranking must account for transitions.

## Worked example
A debt that must be resolved is analyzed not by "will it default" but by ranking the orderly-restructure vs disorderly-crisis routes to the same resolution.
