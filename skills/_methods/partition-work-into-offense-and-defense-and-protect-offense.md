## When to use
Allocating finite attention across a workload mixing edge-generating and commoditized tasks.

## Method
1. Label each activity offense (proprietary work that could create edge) vs defense (consuming the same public flow everyone else does).
2. Recognize defensive work has near-zero marginal advantage no matter how useful it feels standalone — it only keeps you level.
3. Minimize, batch, delegate, or automate defense; spend the bulk of attention on offense.
4. Protect dedicated, interruption-free blocks for offensive work and guard them against reactive intake.

## Inputs → Outputs
- Inputs: full activity inventory; competitor-overlap estimate per activity.
- Outputs: an offense/defense split with a protected offensive schedule.

## Failure modes / assumptions
- Confusing busyness on defense with progress.
- Cutting defense so far that table-stakes coverage gaps create blind spots.

## Worked example (illustrative, single)
A practitioner stops manually re-reading public releases everyone already has (defense) and reallocates the time to a proprietary channel check (offense).
