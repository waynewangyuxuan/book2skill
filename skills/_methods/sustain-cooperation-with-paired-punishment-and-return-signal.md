## When to use
Designing or diagnosing the stability of cooperation in any repeated multi-party interaction (pricing, negotiation, alliances, platform governance).

## Method
1. Install component A — punishment: an immediate, ideally automatic, matching response to any defection so the defector never profits from it.
2. Install component B — return signal: a visible, credible indication of readiness to restore the cooperative state once the defector relents.
3. Verify both are present: punishment without a return signal traps everyone in the bad equilibrium; signaling without credible punishment invites exploitation.
4. Recognize this as the deep structure behind tit-for-tat-style stability; make the punishment cheap/automatic and the return signal unambiguous.

## Inputs → Outputs
- In: a repeated interaction + its current response/signaling mechanisms.
- Out: a diagnosis of which half is missing and a design that supplies both.

## Failure modes / assumptions
- Legal/communication constraints can make the return signal hard to send without crossing collusion lines — it must be conveyed through conduct, not explicit agreement.
- An irrational or relative-performance-driven party may not respond to either component.
- Automatic punishment can over-trigger on noise, sparking unintended escalation.
