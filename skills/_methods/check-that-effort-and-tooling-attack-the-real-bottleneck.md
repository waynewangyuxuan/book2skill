## When to use
When you are tempted to add data collection or model complexity to make progress on a hard problem.

## Method
1. Identify the actual bottleneck that limits the answer's quality.
2. Check whether more data collection would help, or whether the base rate of useful input is so tiny that more volume is wasted.
3. Check whether the hard part is elsewhere (a single unvalidated assumption, an unobservable variable) that tooling cannot touch.
4. Recognize collection/complexity that does not attack the bottleneck as false productivity, and redirect effort to where the bottleneck actually is.

## Inputs to Outputs
- Inputs: the problem; the proposed effort/tooling; the true bottleneck.
- Outputs: effort routed to the bottleneck instead of busywork.

## Failure modes / assumptions
- Adding rows and sources because it feels productive while the bottleneck is untouched.
- Mis-locating the bottleneck.

## Worked example (illustrative, single)
A practitioner stops collecting more low-yield data and instead validates the single assumption that actually gates the conclusion.
